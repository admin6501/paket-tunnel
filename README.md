# GOST Tunnel Manager — تانل ایران ⇄ خارج

تانل پایدار، ضدفیلتر و کم‌مصرف بین **سرور ایران** و **سرور خارج** با موتور [GOST v3].
طراحی‌شده برای اینکه در **ساعات اوج مصرف** نه پینگ بالا برود، نه سرعت افت کند و نه CPU درگیر شود.

## چطور کار می‌کند؟

```
کاربر  ──►  سرور ایران (Entry)  ══[ تانل mwss رمزنگاری‌شده + Mux ]══►  سرور خارج (Exit)  ──►  سرویس اصلی (پنل/SSH)
            پورت محلی باز                  WebSocket + TLS + Multiplex                       127.0.0.1:port
```

- **mwss = WebSocket + TLS + Multiplexing**
  - ترافیک شبیه HTTPS/WebSocket است → عبور بهتر از فیلترینگ
  - Multiplexing چند جریان را روی یک کانکشن جمع می‌کند → **هندشیک و کانکشن کمتر = CPU پایین‌تر** و پایداری بیشتر در شلوغی
- **تیونینگ کرنل**: فعال‌سازی `BBR` + بزرگ‌کردن بافرهای TCP + کاهش تأخیر → حفظ **پینگ و throughput** هنگام ازدحام شبکه
- **systemd**: ری‌استارت خودکار هنگام قطعی و بالا آمدن خودکار بعد از ریبوت (ضد قطعی)

## نصب

### ۱) روی سرور خارج (محل سرویس اصلی):
```bash
sudo bash tunnel.sh
# گزینهٔ 1 را بزن
```
در پایان یک **توکن اتصال** می‌گیری. آن را کپی کن.

### ۲) روی سرور ایران:
```bash
sudo bash tunnel.sh
# گزینهٔ 2 را بزن، آی‌پی سرور خارج و توکن را وارد کن
```

تمام! حالا کاربران به `IP_سرور_ایران:پورت` وصل می‌شوند.

## دستورات سریع (بدون منو)
```bash
sudo bash tunnel.sh foreign     # نصب سرور خارج
sudo bash tunnel.sh iran        # نصب سرور ایران
sudo bash tunnel.sh status      # وضعیت و لاگ
sudo bash tunnel.sh restart     # ری‌استارت
sudo bash tunnel.sh token       # نمایش توکن (سرور خارج)
sudo bash tunnel.sh uninstall   # حذف کامل
```

## فایروال
- سرور خارج: پورت تانل (پیش‌فرض 443) را باز کن → `ufw allow 443/tcp`
- سرور ایران: پورت محلی کاربران را باز کن → `ufw allow <port>/tcp`

## نکات
- اگر `BBR` فعال نشد، کرنل را آپدیت کن (`apt upgrade` و ریبوت). تانل بدون BBR هم کار می‌کند.
- برای ترافیک UDP (مثل WireGuard) هنگام نصب سرور خارج گزینهٔ UDP را `y` بزن.
- ترابری‌های دیگر: `mtls` (سریع‌تر)، `mws` (پشت CDN)، `grpc`.
