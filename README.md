# PerseTunnel — هستهٔ تانل اختصاصی ایران ⇄ خارج

تانل اختصاصیِ نوشته‌شده روی **QUIC** با **استخر چند کانکشنی (Connection Pool)**، مخصوص مسیرهای پُرتأخیر/پُراِفت مثل **آسیاتک (ایران) ↔ آلمان** که زیر **بار سنگین** نه پینگ بالا می‌رود نه سرعت می‌ریزد.

## چرا این بهتر از gRPC/mwss است؟

| مشکل ابزارهای TCP-محور (gRPC/wss) | راه‌حل PerseTunnel |
|---|---|
| TCP-over-TCP meltdown زیر افت بسته | حمل روی **UDP/QUIC** (بدون TCP تو در تو) |
| Head-of-Line blocking (یک بسته گم شه همه قفل) | QUIC هر استریم **flow-control مستقل** دارد |
| گلوگاه تک‌کانکشن زیر بار سنگین | **استخر N کانکشن موازی** + پخش بار روی کم‌بارترین |
| سرعت کم روی RTT بالا | **پنجره‌های بزرگ** (تا ۶۴MB) برای BDP بالا |
| CPU بالا (FEC در KCP) | بدون FEC؛ فقط loss-recovery خود QUIC |

## معماری

```
کاربر ─► سرور ایران (client)  ══╗
                                ║  N سشن موازی QUIC (UDP) + TLS1.3
                                ║  هر کانکشن کاربر → کم‌بارترین سشن
        سرور خارج (server) ◄════╝
              │
              └─► سرویس اصلی (پنل/SSH) روی 127.0.0.1:port
```

- احراز هویت: توکن ۳۲ بایتی مشتق از کلید مشترک (مقایسهٔ constant-time) → ضد پروب فعال.
- استتار: ALPN=h3 و SNI دلخواه → ترافیک شبیه HTTP/3.
- اتصال مجدد خودکار هر سشن + keepalive + systemd Restart=always.

## نصب (خودکفا — Go و سورس داخل اسکریپت است)

### روی سرور خارج (آلمان):
```bash
sudo bash install.sh        # گزینه 1
```
توکن خروجی را کپی کن. **پورت تانل را روی UDP باز کن:** `ufw allow 443/udp`

### روی سرور ایران (آسیاتک):
```bash
sudo bash install.sh        # گزینه 2 → آدرس سرور خارج + توکن + پورت محلی + تعداد کانکشن
```

### دستورهای سریع
```bash
sudo bash install.sh foreign|iran|status|restart|token|uninstall
```

## تنظیم برای بهترین نتیجه
- **`-conns` (تعداد کانکشن استخر):** برای مشتری سنگین مقدار ۴ تا ۸ را امتحان کن. بیشتر = پخش بار بهتر، کمی CPU بیشتر.
- پورت تانل را UDP باز کن (هم ufw هم Security Group پنل ابری).
- اگر اپراتور UDP/QUIC را خیلی محدود کرد، نسخهٔ TCP-محور (`tunnel.sh` با GOST) به‌عنوان جایگزین موجود است.

## نتایج تست (لوکال)
- throughput تک‌جریانی: ~1.3Gbps
- بار سنگین: ۶۰ دانلود همزمان (۶GB) → ۶۰/۶۰ موفق، بدون فروپاشی
- احراز هویت کلید اشتباه → رد

---
### جایگزین TCP-محور
`tunnel.sh` (موتور GOST، ترابری‌های mwss/quic/kcp) هم در همین مخزن هست؛ اگر مسیر شما QUIC خام را محدود می‌کند می‌تواند مفید باشد.

## 🔌 نصب آفلاین (بدون هیچ دانلودی — مخصوص سرور ایران با اینترنت محدود)

اگر سرور نمی‌تواند Go و ماژول‌ها را دانلود کند، از باینریِ از پیش ساخته‌شده استفاده کن.

### روش ۱ (ساده‌ترین): فایل تک، خوداستخراج
سرورها x86_64 هستند → فایل **`dist/persetunnel-offline-amd64.run`** (۸.۴MB) را بردار.
باینری داخلش embed شده؛ فقط همین یک فایل را به سرور منتقل کن:

```bash
# از روی سیستم خودت به سرور:
scp dist/persetunnel-offline-amd64.run root@SERVER_IP:/root/

# روی سرور (هیچ دانلودی نمی‌کند):
sudo bash persetunnel-offline-amd64.run
```
(برای سرور arm64 از `persetunnel-offline-arm64.run` استفاده کن.)

### روش ۲: اسکریپت + باینری جدا
`install-offline.sh` را همراه `dist/persetunnel-linux-amd64` در یک پوشه بگذار:
```bash
sudo bash install-offline.sh            # باینری کنار اسکریپت را پیدا می‌کند
# یا مسیر بده:
sudo bash install-offline.sh --bin /path/to/persetunnel-linux-amd64
```

### نکته
- سرور خارج (آلمان) اینترنت خوبی دارد → می‌تواند همان `install.sh` معمولی را اجرا کند.
- فقط سرور ایران را آفلاین نصب کن. هر دو سر از همان باینری استفاده می‌کنند.
- باینری‌ها **استاتیک** هستند (CGO_ENABLED=0) و هیچ وابستگی سیستمی ندارند.

### فایل‌های آماده در `dist/`
| فایل | توضیح |
|---|---|
| `persetunnel-offline-amd64.run` | نصب‌کنندهٔ تک‌فایل خوداستخراج (x86_64) |
| `persetunnel-offline-arm64.run` | نصب‌کنندهٔ تک‌فایل خوداستخراج (arm64) |
| `persetunnel-linux-amd64` | باینری خام (x86_64) |
| `persetunnel-linux-arm64` | باینری خام (arm64) |
